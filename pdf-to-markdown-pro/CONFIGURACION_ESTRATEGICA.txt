La skill "pdf-to-markdown-pro" ha sido actualizada con la estrategia 
jerárquica de herramientas para máxima fidelidad en conversión de PDF → MD.


📊 JERARQUÍA IMPLEMENTADA
═════════════════════════════════════════════════════════════════════════════

NIVEL 1: SUPERIOR (Prioritario)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ Docling
   • Estructura semántica
   • Tablas complejas
   • Jerarquía de títulos
   • [PRIORITARIO]

✅ MarkItDown (Microsoft)
   • Excelente formato Markdown
   • Jerarquía de títulos
   • [PRIORITARIO]

✅ pymupdf4llm (Optional)
   • Conversión directa PyMuPDF → MD


NIVEL 2: HÍBRIDO (Fallback)
━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ PyMuPDF (importar como fitz)
   • Detección de bloques
   • Extracción de metadatos
   • OCR (si está disponible)

✅ pdfplumber
   • Extracción especializada de tablas
   • Método .extract_table()
   • Superior en alineación visual

✅ tabulate
   • Formateo a Markdown estándar
   • Garantiza formato | | |
   • Preserva relaciones de celdas


NIVEL 3: OCR (Escaneos/Imágenes)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ EasyOCR
   • Para PDFs que son imágenes
   • Sin capa de texto
   • Multiidioma

🎯 REGLAS DE EJECUCIÓN
═════════════════════════════════════════════════════════════════════════════

1. NOMBRES ESTÁNDAR
   ✅ pip install pymupdf pdfplumber docling markitdown
   ❌ NO: pip install pdf-to-markdown-pro

2. IMPORTACIÓN DE PyMuPDF
   ✅ import fitz  (así se importa PyMuPDF)
   ❌ NO: import pymupdf

3. FORMATO DE TABLAS (Obligatorio)
   ✅ | Columna 1 | Columna 2 |
      |-----------|-----------|
      | Dato 1    | Dato 2    |
   
   ✅ Alineación visual preservada
   ✅ Relaciones de celdas intactas

4. FALLBACK AUTOMÁTICO
   Docling → MarkItDown → Híbrido → OCR
